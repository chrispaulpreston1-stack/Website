1. Executive Summary & Threat Level

Threat Level: MEDIUM — Watch closely, not panic. Planiverse is a new entrant in public beta, offering free AI-powered planning decision analysis. It is NOT a competitor to Site Intelligence's core product (submission-ready interpreted reports with 23 datasets), but it IS competing for the same buyer's first touchpoint — the "should I bother?" question. If Planiverse gets product-market fit and scales, they could own the top-of-funnel screening conversation that currently leads to SI's Screening and Feasibility products.

What Planiverse Is

An interactive, AI-powered planning decision analysis tool covering English councils. Users can search a map of planning applications, ask natural language questions about approval rates, get comparable decisions, and generate PDF feasibility reports. Currently in PUBLIC BETA (FREE) with planned pricing of £99/month post-beta.

What Planiverse Is NOT

  • Not a submission-ready report engine (no planning policy analysis, no NPPF clause mapping, no heritage/ecology/flood assessment)
  • Not a multi-discipline platform (planning decisions only — no environmental, geotechnical, transport, or design assessment)
  • Not backed by original data collection (relies entirely on PlanIt.org.uk scraped data + Anthropic Claude AI)
  • Not a replacement for professional planning advice (their own disclaimer says so)

Data Source Assessment: PlanIt.org.uk

PlanIt.org.uk is a free, community-built aggregator run by Andrew Speakman. It scrapes planning data from UK council portals using Python scrapers (UKPlanning library). Key facts:

  • Covers approximately 417 planning authorities
  • ~20 million applications in its database, 91% geocoded
  • Free API available — data dictionary and specs published
  • Data quality depends on scraping reliability — not all fields consistently populated
  • Updated regularly but not real-time — scraping cadence varies by authority
Critical dependency risk: Planiverse's entire data pipeline depends on PlanIt.org.uk — a free, one-person operation. If PlanIt goes down, changes its API, or starts charging, Planiverse loses its data source. Compare this to SI's 21 enrichment modules pulling from multiple paid APIs (OS, EA, BGS, HMLR, etc.) with fallback chains.

Data Flow Architecture

Request flow: User query → Vercel serverless function → PlanIt.org.uk API (planning data) + ArcGIS REST API (boundaries/constraints) → Anthropic Claude API (AI analysis) → Response rendered in vanilla JS → PDFShift (report generation)
  • Pagination: 5,000 items per request with bbox-based spatial filtering
  • Caching: 24-hour server-side cache (mentioned in source), viewport-based constraint caching
  • Stale data prevention: Sequence counters to discard out-of-order API responses
  • GeoJSON: Feature collections for map rendering — no visible client-side validation
  • Session storage: Persists selected council and map centre between page loads (sessionStorage, not localStorage)

Report Verdict

The Planiverse report is a good screening tool — well-designed, visually clear, and useful for the "is this worth pursuing?" question. However, it is fundamentally a statistical analysis of past decisions, not a planning feasibility assessment. It cannot tell you:

  • Whether the site is in a flood zone and what FRA would be required
  • Whether there are protected species or habitats requiring ecology surveys
  • Whether the Local Plan allocates the site for a specific use
  • What CIL/S106 contributions would be due and how they affect viability
  • Whether heritage constraints would trigger pre-app or HIA requirements
  • What the actual planning policy framework says about this type of development here
  • Whether ground conditions require contamination assessment or specific foundations

The NPPF citations in the "Key Risks" section are particularly risky — Claude's training data may produce plausible but incorrect paragraph references, especially post-December 2024 NPPF revision.

11. Planiverse vs Site Intelligence: Head-to-Head

Planiverse Advantages Over SI

  • Free (currently) — zero barrier to entry
  • Interactive map explorer — visual, explorable, not a static PDF
  • Instant results — no waiting for report generation
  • Natural language chat — ask anything about planning in an area
  • Site-finding capability — "where should I build?" question SI doesn't answer
  • Live pipeline tracking — competitor monitoring SI doesn't offer
  • Comparable decisions mapped — spatial visualisation of precedents
  • Per-element breakdown — ward approval rates by proposal element is novel
  • Decision speed distribution — practical metric SI doesn't include
  • Subscription model — unlimited use, good for frequent users
  • Transparency — every stat links to original council record

SI Advantages Over Planiverse

  • 23 data sources vs Planiverse's 1 (PlanIt only)
  • Submission-ready reports — not just screening analysis
  • Planning policy analysis — actual Local Plan and NPPF assessment
  • 650 verified appeal cases vs AI-generated NPPF citations
  • Environmental assessment — flood, ecology, geology, contamination
  • Heritage assessment — listed buildings, conservation areas, scheduled monuments
  • CIL/S106 calculations — 178 LPAs modelled
  • Viability assessment — GDV, costs, margins
  • Transport / highways — access, parking, trip generation
  • 17-layer QA pipeline — not relying on AI hallucination alone
  • 125 specialised agents vs one general-purpose Claude chat
  • 40+ report types including summaries for different audiences
  • Professional credibility — reports can be submitted with applications
  • Cross-domain intelligence — cost cascades, programme risks, design compression

The Critical Question: Are They Competing for the Same Customer?

Partially, yes. Both target small developers and independent architects asking "should I pursue this site?" But they answer different questions at different depths:
  • Planiverse answers: "Based on past decisions nearby, what are my statistical odds?" (screening question)
  • SI answers: "Based on every relevant constraint, policy, and precedent, what are the actual planning risks and how do I mitigate them?" (feasibility question)

The danger is not that Planiverse replaces SI. The danger is that a developer uses Planiverse's free tool, gets a "75% approval rate" figure, and decides that's enough to proceed — skipping the SI Screening or Feasibility product entirely. Planiverse could capture the intent before SI sees it.

13. Strategic Implications for Site Intelligence

What Planiverse Gets Right (and SI Should Learn From)

  1. The "Will the council approve it?" framing — simple, buyer-centric question. SI should use this language in marketing.
  2. Interactive map explorer — this is compelling UX that SI's static PDF model lacks. Consider an interactive screening tool as a lead-gen funnel.
  3. Comparable decisions visualisation — mapping nearby approvals/refusals spatially is more powerful than listing them. SI's reports could benefit from this presentation.
  4. Per-element approval breakdown — showing ward-specific rates for "rear extension" vs "loft conversion" vs "new dwelling" is genuinely useful granularity.
  5. Decision speed metrics — practical information SI doesn't currently include. Worth adding to the LPA analysis section.
  6. Transparency claims — "every stat links to the original record" is a powerful trust signal. SI should emphasise its source provenance more prominently.
  7. Free entry point — Planiverse is training users to expect free planning intelligence. SI needs a free or very low-cost entry point to compete for attention.

Recommended SI Responses

Defensive Actions (Short-term)

  • Add comparable decision statistics to SI Screening reports — approval rates by type within the ward. This data is available via PlanIt API or planning.data.gov.uk. Low effort, high impact.
  • Add LPA decision speed metrics — average determination time by application type. Available from the same data sources.
  • Emphasise depth advantage in marketing — "23 datasets vs 1" and "submission-ready vs screening only" messaging.
  • Create a free planning risk check — even a simple "enter your postcode, see your constraints" tool would capture the top-of-funnel intent that Planiverse is targeting.

Offensive Actions (Medium-term)

  • Build an interactive site screening tool on the website — map-based, constraint-aware, free to use. Capture the email. Upsell to Screening (£199) and Feasibility (from £695).
  • Publish planning approval rate content — per-council, per-ward approval rates as SEO content. This is the exact traffic Planiverse is trying to capture.
  • Position against Planiverse directly — "Statistical odds are not planning advice. Here's what approval rates don't tell you." (blog post / LinkedIn content)
  • Partner with PlanIt.org.uk — integrate comparable decision data into SI reports. Add the interactive element Planiverse has, but backed by SI's 23-dataset depth.

What NOT to Worry About

  • Planiverse replacing SI's core product: It cannot. It has no policy analysis, no environmental data, no viability, no submission-ready output. The gap is enormous.
  • Price competition: At £99/month vs SI's £695+ per report, they serve different needs. Nobody is choosing between them for the same purchase.
  • Scale threat: Solo founder, no funding, no team, no press. This is not a well-resourced competitor. It could die quietly or get acqui-hired.

The Real Risk

The real risk is not Planiverse specifically — it's the category. Free AI planning tools are inevitable. If not Planiverse, then PlanningLens, PlanningHub, or someone else will own the "quick free check" layer. SI needs to either own that layer (with a free screening tool) or ensure its paid products are so clearly superior that the free tools serve as a marketing funnel rather than a substitute. The worst outcome is a developer using Planiverse, getting "78% approval rate", and never discovering that their site has a contaminated land designation, a Grade II listed building within the setting, and a £45,000 CIL liability.
